## The one-line pitch

I build the layer between "we have a pile of unstructured data" and "here is a ranked, explainable decision" — using retrieval, embeddings, and LLMs, wired into real backend systems instead of notebooks.

<br>

## What I'm building right now

**AI-Powered Global Talent Research & Recommendation Engine** — an enterprise recruitment platform that reads resumes the way a human recruiter would, then does what a human can't: rank, compare, and surface skill gaps across thousands of candidates in seconds.

| Layer | What it does |
|---|---|
| 🧾 Resume understanding | Parses unstructured resumes into structured candidate profiles |
| 🔎 Semantic search | Vector-based matching between candidates, roles, and skills |
| 📊 Ranking | Scores and orders candidates against a role's real requirements |
| 🧩 Skill gap analysis | Flags what a candidate is missing for a target role |
| 💬 RAG knowledge layer | Lets recruiters *ask questions* about candidates instead of filtering |

This isn't a side project — it's the core of my current AI/Data Science internship work on enterprise recruitment automation.
